ZonalStats

Creating a ZonalStats object

ZonalStats() is the main object to request zonal statistics using the Google Earth Engine Python API. The object can be initialized with the following parameters.

class gee_tools.ZonalStats(target_features, statistic_type, collection_id, ee_dataset=None, band=None, output_name=None, output_dir=None, frequency='original', temporal_stat=None, scale=250, min_threshold=None, mask=None, tile_scale=1, start_year=None, end_year=None, scale_factor=None)

Python class to calculate zonal and temporal statistics from Earth Engine datasets (ee.ImageCollection or ee.Image) over vector shapes (ee.FeatureCollections).

Parameters:

  • target_features (ee.FeatureCollection) – vector features

  • statistic_type (str - mean, max, median, min, sum, stddev, var, count, minmax, p75, p25, p95, all) – method to aggregate image pixels by zone

  • collection_id (str) – ID for Earth Engine dataset

  • ee_dataset (ee.Image or ee.ImageCollection(, optional)) – input dataset if no collection ID is provided, defaults to None

  • band (str(, optional)) – name of image band to use

  • output_name (str) – file name for output statistics if saved to Google Drive, defaults to None

  • output_dir (str(, optional)) – directory name for output statistics if saved to Google Drive, defaults to None

  • frequency (str (monthly, annual, or original)(, optional)) – temporal frequency for aggregation, defaults to original frequency

  • temporal_stat (str (mean, max, median, min, sum)(, optional)) – statistic for temporal aggregation, defaults to None.

  • scale (int) – scale for calculation in mts

  • min_threshold (int) – filter out values lower than treshold

  • mask (ee.Image) – filter out observations where mask is zero

  • tile_scale (int) – tile scale factor for parallel processing, defaults to 1

  • start_year (int) – specify start year for stats, defaults to None

  • end_year (int) – specify end year for stats (not inclusive), defaults to None

  • scale_factor (int) – scale factor to multiply ee.Image to get correct units, defaults to None

A shapefile can be called directly from GEE assets, or easily converted using gee_helpers.gpd_to_gee or the geemap package.

Retrieving output table

1. Retrieve output table directly

Statistics can be accessed as the result of ZonalStats.runZonalStats() This will be computed within the python earth engine environment.

from gee_tools import ZonalStats
AOIs = ee.FeatureCollection('users/afche18/Ethiopia_AOI') # ID of ee.FeatureCollection
zs = ZonalStats(
  collection_id = 'LANDSAT/LC08/C01/T1_8DAY_NDVI',
  target_features = AOIs,
  statistic_type = "mean",
  frequency = "annual",
  temporal_stat = "mean"
)
df = zs.runZonalStats()

2. Submit an EE Task

Alternatively, a task can be submitted to the Earth Engine servers by specifying an output_name and outuput_dir.

This option is recommended to run statistics for big areas or for a high number of collections. The output table will be saved on the specified directory in Google Drive.

import ee
from gee_tools import ZonalStats
ee.Initialize()
zs = ZonalStats(
  collection_id = 'LANDSAT/LC08/C01/T1_8DAY_NDVI',
  target_features = AOIs,
  statistic_type = "mean",
  frequency = "annual",
  temporal_stat = "mean,
  output_dir="pretty_folder",
  output_name="pretty_file"
)
zs.runZonalStats()

The status of the task can be monitored with ZonalStats.reportRunTime()

>>> zs.reportRunTime()
Completed
